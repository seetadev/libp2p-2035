# Contributing to libp2p-2035

We welcome:
- Research proposals
- Formal models
- Simulation benchmarks
- Overlay prototypes
- Critical architectural discussions

## Process

1. Open an issue under the appropriate track.
2. Submit RFC-style pull requests for major proposals.
3. Include evaluation metrics where applicable.

## Evaluation Criteria

- Latency
- Message overhead
- Churn resilience
- Sybil resistance
- Partition healing
- Backward compatibility
